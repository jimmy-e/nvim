# Plan: Fix nvim-tree Cmd+1 behavior

## Problem
- Current: tree opens WITH focus on startup, focus trapped in tree
- Desired: tree OPEN but focus in EDITOR; Cmd+1 takes focus TO tree

## Solution

1. **VimEnter autocmd** (`lua/plugins/navigation.lua`): change from `focus = true` to `focus = false`
   - Tree opens visible but cursor stays in editor

2. **Cmd+1 keymap** (`lua/keymaps.lua`): keep as open/focus (never close)
   - If tree closed: open with focus
   - If tree already open: just focus it
